capabilities via the CPUID instruction. Its meant to be a single reference 
for all the enums, helper functions, and feature flags.

1. OVERVIEW & DESIGN
--------------------

The whole thing is a single header-only C library. No .c files to compile, 
just #include "cpuid.h" and your good to go. Everything is static inline 
so the compiler can optimise it away completely when not used.

I wrote this because I got tired of copy-pasting cpuid snippets from 
StackOverflow and Intel's manuals. This .h file has:

  - One enum (cpu_has_query_t) that lists everything you might want to know
  - One function (cpu_has()) that returns the value for any query
  - A bunch of specialised helpers for common tasks (cache sizes, topology, etc)

The library assumes GCC/Clang style inline assembly. It wont work on MSVC 
without modification (you'd need __cpuid intrinsics instead).

2. DATA STRUCTURES
------------------

typedef struct {
    uint32_t eax, ebx, ecx, edx;
} cpuid_res_t;

This is the raw result from a CPUID instruction. All four registers packed 
into a nice little struct. Most of the time you wont touch this directly 
but its there if you need to do custom parsing.


3. CORE INLINE FUNCTIONS
------------------------

3.1 cpuid_query(leaf, subleaf)
    The lowest-level function. Executes CPUID with the given leaf and 
    subleaf values and returns a cpuid_res_t. This is where the actual 
    asm block lives.

    Example:
        cpuid_res_t r = cpuid_query(1, 0);
        if (r.edx & (1 << 25)) printf("SSE is supported\n");

3.2 cpuid_leaf_supported(leaf)
    Checks wether a given leaf is valid on this CPU. For standard leaves 
    (0x00000000 - 0x7FFFFFFF), it queries leaf 0 and compares against EAX. 
    For extended leaves (0x80000000+), it queries 0x80000000 and compares. 
    Leaf 0 is always assumed valid.

    This is used internally by almost everything so you dont have to worry 
    about invalid leaves causing problems.


4. CPUID LEAF CONSTANTS
-----------------------

These are the leaf values you can pass to cpuid_query(). I grouped them 
by category for easier reading.

Standard Leaves (0x00000000 - 0x0000001F):
  CPUID_VENDOR_STRING             0x00000000   - Vendor ID string (EBX,EDX,ECX)
  CPUID_FEATURES                  0x00000001   - Basic feature flags
  CPUID_CACHE_TLB_INFO            0x00000002   - Legacy cache/TLB desciptors
  CPUID_PROCESSOR_SERIAL          0x00000003   - Processor serial number (Pentium III only, mostly useless now)
  CPUID_DETERMINISTIC_CACHE       0x00000004   - Modern cache enumeration (preferred over leaf 2)
  CPUID_MONITOR_MWAIT             0x00000005   - MONITOR/MWAIT parameters
  CPUID_THERMAL_POWER_MGMT        0x00000006   - Thermal and power management features
  CPUID_STRUCTURED_FEATURES       0x00000007   - Extended features (AVX2, AVX-512, etc)
  CPUID_DIRECT_CACHE_ACCESS       0x00000009   - DCA (Data Center stuff)
  CPUID_ARCH_PERF_MON             0x0000000A   - Architectural perf monitoring
  CPUID_EXTENDED_TOPOLOGY         0x0000000B   - Topology enumeration v1
  CPUID_XSAVE_FEATURES            0x0000000D   - XSAVE area sizes and offsets
  CPUID_RDT_MONITORING            0x0000000F   - Resource monitoring (CMT, MBM)
  CPUID_RDT_ALLOCATION            0x00000010   - Resource allocation (CAT, CDP, MBA)
  CPUID_SGX_CAPABILITIES          0x00000012   - Intel SGX info
  CPUID_INTEL_TRACE               0x00000014   - Intel Processor Trace
  CPUID_TSC_INFO                  0x00000015   - TSC/core crystal clock ratio
  CPUID_PROCESSOR_FREQUENCY       0x00000016   - Base/max/bus frequency in MHz
  CPUID_SOC_VENDOR                0x00000017   - SoC vendor info
  CPUID_HYBRID_INFO               0x0000001A   - Hybrid CPU core type info (Alder Lake+)
  CPUID_PCONFIG_INFO              0x0000001B   - Platform configuration
  CPUID_V2_EXTENDED_TOPOLOGY      0x0000001F   - Topology enumeration v2

Hypervisor Leaves (0x40000000 - 0x40000001):
  CPUID_HYPERVISOR_INTERFACE      0x40000000   - Hypervisor vendor string
  CPUID_HYPERVISOR_FEATURES       0x40000001   - Hypervisor feature flags

Extended Leaves (0x80000000 - 0x80000021):
  CPUID_EXT_MAX_FUNCTION          0x80000000   - Max extended leaf
  CPUID_EXT_FEATURES              0x80000001   - Extended feature flags (AMD)
  CPUID_EXT_BRAND_STRING_1        0x80000002   - Part 1 of 48-byte brand string
  CPUID_EXT_BRAND_STRING_2        0x80000003   - Part 2 of 48-byte brand string
  CPUID_EXT_BRAND_STRING_3        0x80000004   - Part 3 of 48-byte brand string
  CPUID_EXT_L1_CACHE_TLB          0x80000005   - L1 cache/TLB info (AMD)
  CPUID_EXT_L2_L3_CACHE_TLB       0x80000006   - L2/L3 cache info (AMD)
  CPUID_EXT_ADV_POWER_MGMT        0x80000007   - Advanced power management
  CPUID_EXT_ADDR_SIZE             0x80000008   - Physical/virtual address sizes
  CPUID_EXT_SVM_FEATURES          0x8000000A   - AMD SVM (virtualization)
  CPUID_EXT_PERF_OPTIMIZATIONS    0x8000001A   - Perf optimizations
  CPUID_EXT_INSTRUCTION_SET       0x8000001B   - Instruction set features
  CPUID_EXT_LWP                   0x8000001C   - Light Weight Profiling
  CPUID_EXT_CACHE_PROPERTIES      0x8000001D   - Cache properties (AMD equivalent of leaf 4)
  CPUID_EXT_TOPOLOGY              0x8000001E   - Topology (AMD)
  CPUID_EXT_MEM_ENCRYPTION        0x8000001F   - Memory encryption (SEV)
  CPUID_EXT_PASID_FEATURES        0x80000020   - PASID features
  CPUID_EXT_SVM_FEATURES_V2       0x80000021   - SVM features v2

    STEPPING_ID, MODEL_ID, FAMILY_ID, PROCESSOR_TYPE,
    EXTENDED_MODEL_ID, EXTENDED_FAMILY_ID,
    LOGICAL_PROCESSORS_PER_PACKAGE, INITIAL_APIC_ID,
    BASE_FREQUENCY_MHZ, MAX_FREQUENCY_MHZ, BUS_FREQUENCY_MHZ,
    PHYSICAL_ADDRESS_BITS, VIRTUAL_ADDRESS_BITS,
    PMU_VERSION, PMU_GP_COUNTERS, PMU_COUNTER_BITS,
    MAX_STANDARD_LEAF, MAX_EXTENDED_LEAF,
    IS_HYBRID, CORE_TYPE, NATIVE_MODEL_ID,

    /* String queries - return const char* (thread-local buffer) */
    VENDOR_STRING, BRAND_STRING, MICROARCHITECTURE,

    CPU_HAS_QUERY_COUNT
} cpu_has_query_t;

Usage examples:

    if (cpu_has(AVX_SUPPORT)) {
        // Use AVX instructions
    }

    int cores = cpu_has(LOGICAL_PROCESSORS_PER_PACKAGE);
    printf("Logical processors: %d\n", cores);

    const char *vendor = (const char*)cpu_has(VENDOR_STRING);
    printf("Vendor: %s\n", vendor);

    const char *uarch = (const char*)cpu_has(MICROARCHITECTURE);
    printf("Microarchitecture: %s\n", uarch);

The MICROARCHITECTURE query is particularly handy. It tries to identify 
the exact CPU generation based on family/model/stepping. Currently supports:

Intel:
  - Atom: Bonnell, Silvermont, Airmont, Goldmont, Tremont, Gracemont
  - Core: Nehalem, Westmere, Sandy Bridge, Ivy Bridge, Haswell, Broadwell,
          Skylake, Kaby Lake/Coffee Lake, Comet Lake, Ice Lake, Tiger Lake,
          Alder Lake, Raptor Lake, Meteor Lake, Lunar Lake
  - Xeon Phi: Knights Landing/Mill
  - Old stuff: NetBurst (Pentium 4)

AMD:
  - K8, K10, Bulldozer family, Zen family, Zen 4/5, Hygon Dhyana

If it cant identify the CPU, it returns "Unknown Intel", "Unknown AMD", 
or "Unknown Vendor" depending on the vendor string.


7. ENUMERATION & ITERATION HELPERS
----------------------------------

7.1 cpuid_enum_subleaf(leaf, &subleaf, &out)
    Iterates over all valid subleaves for a given leaf. You initialise 
    subleaf to 0, then call this in a loop. It returns true while theres 
    more data, false when done. It automatically increments subleaf.

    Supported leaves for enumeration:
      - CPUID_DETERMINISTIC_CACHE (0x04): enumerates all cache levels
      - CPUID_EXT_CACHE_PROPERTIES (0x8000001D): AMD cache enumeration
      - CPUID_EXTENDED_TOPOLOGY (0x0B): topology levels
      - CPUID_V2_EXTENDED_TOPOLOGY (0x1F): topology v2
      - CPUID_XSAVE_FEATURES (0x0D): XSAVE components 0-63
      - CPUID_STRUCTURED_FEATURES (0x07): all subleaves
      - CPUID_SGX_CAPABILITIES (0x12): subleaves 0-2
      - CPUID_INTEL_TRACE (0x14): subleaves 0-1
      - CPUID_SOC_VENDOR (0x17): all subleaves

    Example - enumerate caches:
        uint32_t sub = 0;
        cpuid_res_t res;
        while (cpuid_enum_subleaf(CPUID_DETERMINISTIC_CACHE, &sub, &res)) {
            // process cache info
        }

7.2 cpuid_display_family()
    Returns the "display family" as per Intel/AMD convention. This is 
    family + extended_family when family == 0x0F, otherwise just family.

7.3 cpuid_display_model()
    Returns the "display model". This is (extended_model << 4) | model 
    when family is 0x06 or 0x0F, otherwise just model.


8. CACHE INFORMATION HELPERS
----------------------------

8.1 cpuid_cache_info(subleaf, &level, &type, &size_kb)
    Queries a specific cache subleaf and extracts the level (1, 2, 3), 
    type (1=data, 2=instruction, 3=unified), and size in KB.

    Returns false if the subleaf is invalid.

8.2 cpuid_cache_shared_threads(subleaf)
    Returns how many logical processors share this cache. This is useful 
    for figuring out cache topology.

8.3 cpuid_num_caches()
    Returns the total number of cache levels reported by the CPU. 
    Just iterates through all valid subleaves of leaf 0x04 and counts them.

    Example:
        uint32_t n = cpuid_num_caches();
        printf("This CPU reports %u cache entries\n", n);


9. XSAVE / EXTENDED STATE HELPERS
---------------------------------

9.1 cpuid_xsave_component_supported(component)
    Checks if a specific XSAVE component (0-63) is supported. Queries 
    leaf 0x0D with the component number as subleaf. Returns true if 
    either EAX or EBX is non-zero.

9.2 cpuid_xsave_component_size(component)
    Returns the size in bytes of the save area for this component.

9.3 cpuid_xsave_component_offset(component)
    Returns the offset in the XSAVE area where this component starts.

    These are usefull if your writing an OS or hypervisor that needs to 
    manage XSAVE state manually. Most application code can just use 
    the XGETBV instruction via compiler intrinsics.


10. RDT (RESOURCE DIRECTOR TECHNOLOGY) HELPERS
-----------------------------------------------

10.1 cpuid_rdt_monitoring_supported(resource)
     Checks if a specific monitoring resource is available. The resource 
     parameter goes into ECX (subleaf). Returns true if EDX[4:0] is non-zero.

10.2 cpuid_rdt_allocation_supported(resource)
     Checks if a specific allocation resource is available. Returns true 
     if EAX[4:0] is non-zero.

     RDT is mostly relevant for data centers and cloud providers who want 
     to enforce cache/memory bandwidth QoS. If your just writing normal 
     software you probably dont need this.


11. SGX HELPERS
---------------

11.1 cpuid_sgx_info(&sgx1, &sgx2, &max_enclave_64, &max_enclave_32)
     Queries SGX capabilities. Sets sgx1 and sgx2 booleans, and returns 
     the maximum enclave size as a power-of-two exponent for 64-bit and 
     32-bit modes.

     For example, if max_enclave_64 is 36, the max enclave size is 2^36 
     bytes (64 GB).

     Note: SGX has been deprecated on newer Intel consumer CPUs but is 
     still present on some Xeons. Also, just because the CPU supports it 
     doesnt mean the BIOS has enabled it.


12. TOPOLOGY HELPERS
--------------------

12.1 cpuid_topology_level(subleaf, &level_type, &num_logical)
     Queries leaf 0x0B (extended topology) and returns the level type 
     and number of logical processors at that level.

     Level types:
       1 = SMT (hyperthread)
       2 = Core
       3 = Module (AMD)
       4+ = Higher levels (die, package, etc)

     Returns false if the subleaf is invalid (ECX[15:8] == 0).


13. VENDOR & BRAND STRING HELPERS
---------------------------------

13.1 __cpuid_get_vendor(buf)
     Internal function that fills a 13-byte buffer with the vendor string. 
     Usually "GenuineIntel" or "AuthenticAMD". Also seen: "CentaurHauls", 
     "CyrixInstead", "TransmetaCPU", "GenuineTMx86", "Geode by NSC", 
     "NexGenDriven", "RiseRiseRise", "SiS SiS SiS ", "UMC UMC UMC ", 
     "VIA VIA VIA ", "Vortex86 SoC", "  Shanghai  " (Zhaoxin), etc.

13.2 __cpuid_get_brand(buf)
     Internal function that fills a 49-byte buffer with the processor 
     brand string from leaves 0x80000002-0x80000004. Also strips leading 
     spaces because Intel likes to pad with spaces for some reason.

     Note: Some older CPUs dont support extended leaves, in which case 
     the buffer will be empty (just a null terminator).

13.3 cpuid_hypervisor_vendor(buf)
     If running under a hypervisor (HYPERVISOR_PRESENT is true), fills 
     a 13-byte buffer with the hypervisor vendor string. Common values:
       "VMwareVMware" - VMware
       "Microsoft Hv" - Hyper-V
       "KVMKVMKVM"    - KVM
       "XenVMMXenVMM" - Xen
       "prl hyperv  " - Parallels
       "VBoxVBoxVBox" - VirtualBox

     Returns false if not running under a hypervisor.


14. FREQUENCY & TIMING HELPERS
------------------------------

14.1 cpuid_tsc_frequency_hz()
     Attempts to calculate the TSC frequency from leaf 0x15. Returns 
     (crystal_hz * numer) / denom. If crystal_hz is zero, assumes 
     24 MHz for Intel CPUs (which is the common case).

     Returns 0 if leaf 0x15 is unsupported or if denom/numer are zero.

     Note: This gives the nominal TSC frequency, not the current 
     operating frequency. For current frequency you need MSR reads 
     or OS APIs.

14.2 BASE_FREQUENCY_MHZ, MAX_FREQUENCY_MHZ, BUS_FREQUENCY_MHZ
     These are cpu_has() queries that read leaf 0x16. Not all CPUs 
     support this leaf (its relatively recent). Returns 0 if unsupported.


15. HYPERVISOR DETECTION
------------------------

The HYPERVISOR_PRESENT query checks bit 31 of leaf 1 ECX. This is the 
official way to detect if your running virtualised. However, some 
hypervisors (looking at you, older VMware versions) didnt set this bit 
properly. If you need more robust detection, combine it with:

  - Checking for hypervisor-specific CPUID leaves (0x40000000+)
  - Timing attacks (TSC behaviour differs under virt)
  - Checking known hypervisor signatures in DMI/ACPI tables

For most purposes though, HYPERVISOR_PRESENT + cpuid_hypervisor_vendor() 
is good enough.


16. KNOWN ISSUES / TODOS
------------------------

- The leaf 0x07 EDX feature bits have some collisions (same bit number 
  used for different features on different subleaves). The header defines 
  them all on subleaf 0 which is technically incorrect for some. Need to 
  figure out a cleaner way to handle subleaf-specific bits.

- MICROARCHITECTURE detection is incomplete. Missing lots of older CPUs 
  (Pentium M, Core Duo, various AMD K7s, etc) and very new ones might 
  not be in there yet. I add them if i need to support them, you can make a PR if you need one.

- No support for ARM or other architectures. This is x86/x64 only by 
  design. If you need ARM CPU feature detection, look at HWCAP on Linux 
  or IsProcessorFeaturePresent on Windows.

- The __cpuid_check_feature and __cpuid_extract_field functions are 
  technically internal (double underscore prefix) but their exposed in 
  the header. Might wrap them better in a future version.

- Thread-local storage for string buffers uses static __thread which 
  is GCC/Clang specific. MSVC would need __declspec(thread).

- No caching of results. Every call to cpu_has() re-executes CPUID. 
  For a feature check in a hot loop this could matter. In practice 
  CPUID is fast (hundreds of cycles) but if your calling it millions 
  of times you should cache the result yourself.

- The cpuid_enum_subleaf() function for CPUID_STRUCTURED_FEATURES (0x07) 
  has a subtle bug: it reads the max subleaf from EAX on the first call, 
  but if you start with subleaf != 0 it wont know the max. Always start 
  at 0.

- Missing AMX (Advanced Matrix Extensions) detailed queries beyond the 
  basic feature bits. Leaf 0x1D and 0x1E have AMX tile info that isnt 
  wrapped yet.

- Would be nice to add a cpuid_get_cache_size_by_level(level, type) 
  helper that searches through the enumerated caches instead of requiring 
  the caller to iterate manually.

BUILDING & USING
----------------

Just include the header:

    #include "cpuid.h"

Compile with any GCC/Clang based toolchain. Requires:
  - stdint.h, stdbool.h, string.h
  - GCC-style inline assembly (x86 or x86-64)
  - C99 or later

Example program:

    #include <stdio.h>
    #include "cpuid.h"

    int main(void) {
        printf("Vendor: %s\n", (const char*)cpu_has(VENDOR_STRING));
        printf("Brand:  %s\n", (const char*)cpu_has(BRAND_STRING));
        printf("AVX2:   %s\n", cpu_has(AVX2_SUPPORT) ? "yes" : "no");
        printf("Cores:  %lu\n", (unsigned long)cpu_has(LOGICAL_PROCESSORS_PER_PACKAGE));
        return 0;
    }

Compile:
    gcc -O2 -o cpuid_test cpuid_test.c
